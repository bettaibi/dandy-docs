# Input

A flexible text input component with support for multiple variants, sizes, validation states, and icon slots. Provides consistent styling across light and dark themes with mobile-first design.

## Installation & Integration

### Import

```tsx
import { Input } from "dandy-ui";
```

## Props

| Prop         | Type                                             | Default   | Description                                  |
| ------------ | ------------------------------------------------ | --------- | -------------------------------------------- |
| variant      | `default \| success \| error \| warning \| info` | `default` | Visual style variant with semantic colors    |
| size         | `sm \| md \| lg`                                 | `md`      | Input height (sm: 32px, md: 40px, lg: 48px)  |
| rounded      | `sm \| md \| lg \| xl \| full \| none`           | -         | Border radius size                           |
| errorMessage | `string`                                         | -         | Error message shown in tooltip on hover      |
| leftSlot     | `React.ReactNode`                                | -         | Content rendered on the left (icons, badges) |
| rightSlot    | `React.ReactNode`                                | -         | Content rendered on the right (icons, etc.)  |
| disabled     | `boolean`                                        | `false`   | Disables the input                           |
| readOnly     | `boolean`                                        | `false`   | Makes the input read-only                    |
| className    | `string`                                         | -         | Additional CSS classes                       |

Inherits all standard HTML input attributes (`onChange`, `onFocus`, `onBlur`, `value`, `defaultValue`, `placeholder`, `type`, etc.).

## Accessibility

- Keyboard navigable with Tab key
- Focus visible with semantic color ring indicators
- WCAG 2.1 Level AA compliant color contrast
- Minimum 32px height on small size for mobile touch targets
- Error messages accessible via InfoTip tooltip on hover/focus
- Supports all native input types and HTML attributes
- Disabled and read-only states properly communicated
