# Radio

A customizable radio button component built on native HTML input elements with theme-aware styling, multiple variants, and automatic error handling.

## Installation & Integration

### Import

```tsx
import { Radio } from "dandy-ui";
```

## Props

| Prop         | Type                                             | Default   | Description                               |
| ------------ | ------------------------------------------------ | --------- | ----------------------------------------- |
| variant      | `default \| success \| error \| warning \| info` | `default` | Visual style variant with colored borders |
| size         | `sm \| md \| lg`                                 | `md`      | Radio button size                         |
| errorMessage | `string`                                         | `""`      | Error message shown in InfoTip on hover   |
| className    | `string`                                         | -         | Additional CSS classes                    |

Inherits all standard HTML input attributes (`onChange`, `checked`, `name`, `value`, etc.).

## Accessibility

- Native radio input for full keyboard support
- Focus visible with colored ring indicator matching variant
- WCAG 2.1 Level AA compliant
- Screen reader compatible with proper semantic elements
- Respects `prefers-reduced-motion` for animations
- Sufficient color contrast in all states
- Clear focus indicators for keyboard navigation
