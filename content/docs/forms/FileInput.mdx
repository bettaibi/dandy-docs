# FileInput

A production-ready file input component with built-in validation, file metadata display, and clear functionality. Replaces the browser's default file input with a consistent, accessible interface.

## Installation & Integration

### Import

```tsx
import { FileInput } from "dandy-ui";
```

## Props

| Prop          | Type                                             | Default   | Description                                    |
| ------------- | ------------------------------------------------ | --------- | ---------------------------------------------- |
| variant       | `default \| success \| error \| warning \| info` | `default` | Controls border and focus colors               |
| size          | `sm \| md \| lg`                                 | `md`      | Component size                                 |
| rounded       | `sm \| md \| lg \| xl \| full \| none`           | -         | Border radius override                         |
| errorMessage  | `string`                                         | `""`      | Error message displayed in tooltip             |
| placeholder   | `string`                                         | -         | Placeholder text when no file is selected      |
| onFilesChange | `(files: File[]) => void`                        | -         | Callback when files are selected or changed    |
| showMetadata  | `boolean`                                        | `true`    | Show file size and metadata in input           |
| disabled      | `boolean`                                        | `false`   | Disables the file input                        |
| multiple      | `boolean`                                        | `false`   | Allow selecting multiple files                 |
| accept        | `string`                                         | -         | Accepted file types (e.g., "image/\*", ".pdf") |
| className     | `string`                                         | -         | Additional CSS classes                         |

Inherits all standard HTML input attributes (`id`, `name`, `required`, etc.).

## Accessibility

- Keyboard navigable with Tab and Enter/Space keys
- Focus visible with ring indicator
- WCAG 2.1 Level AA compliant
- Minimum 44x44px touch targets for mobile
- Screen reader compatible with proper ARIA labels
- Error state communicated via `aria-invalid` attribute
- Clear button has descriptive `aria-label`
- Browse button includes proper semantic button element
