# Section

A semantic section wrapper that provides consistent vertical spacing. Supports polymorphic rendering via the `as` prop and optional horizontal centering for page-level layout composition.

## Installation & Integration

### Import

```tsx
import { Section } from "dandy-ui";
```

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| spacing | `"none" \| "xs" \| "sm" \| "md" \| "lg" \| "xl" \| "2xl" \| "3xl"` | `"md"` | Vertical spacing size (scales responsively) |
| as | `"section" \| "div" \| "article" \| "aside" \| "main" \| "header" \| "footer"` | `"section"` | Render as a different semantic element |
| centered | `boolean` | `false` | Whether to center content horizontally |
| hidden | `"sm" \| "md" \| "lg" \| "xl"` | - | Hide content at the specified breakpoint and above |
| className | `string` | - | Additional CSS classes |

Inherits all standard HTML element attributes (except `hidden`).

## Accessibility

- Renders as a semantic `<section>` element by default for proper document structure
- Supports polymorphic rendering to use correct landmark elements (`main`, `aside`, `header`, `footer`, `article`)
- Screen readers recognize landmark roles automatically from semantic elements
- Use an `aria-label` or heading child to provide a meaningful section label
