# Badge

A versatile badge component for displaying status indicators, notification counts, categorical labels, and other compact information. Supports multiple visual styles and theme-based coloring.

## Installation & Integration

### Import

```tsx
import { Badge } from "dandy-ui";
```

## Props

| Prop       | Type                                                                               | Default   | Description                                            |
| ---------- | ---------------------------------------------------------------------------------- | --------- | ------------------------------------------------------ |
| variant    | `default \| primary \| secondary \| accent \| success \| warning \| error \| info` | `default` | Badge color theme                                      |
| size       | `sm \| md \| lg`                                                                   | `md`      | Badge size (mobile-first)                              |
| appearance | `solid \| outline`                                                                 | `solid`   | Visual style (filled or bordered)                      |
| shape      | `default \| pill`                                                                  | `default` | Shape style (rounded corners or fully rounded)         |
| children   | `React.ReactNode`                                                                  | -         | Badge content (required if label not provided)         |
| label      | `string`                                                                           | -         | Badge text content (required if children not provided) |
| className  | `string`                                                                           | -         | Additional CSS classes                                 |

Inherits all standard HTML `span` attributes (`onClick`, `role`, `aria-*`, etc.).

## Accessibility

- Uses semantic `span` element for static content
- Color is not the sole indicator of meaning; text content provides context
- If used as a live status indicator, add `role="status"` or `aria-live` attributes
- Focus visible with ring indicator for interactive badges
- Selectable text disabled by default (`select-none`)
- Screen reader compatible with proper semantic structure
